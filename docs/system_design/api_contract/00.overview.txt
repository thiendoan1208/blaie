API Contract - Blaie MVP V1 - Overview

Tai lieu nay quy dinh cac quy uoc chung ap dung cho toan bo API cua du an Blaie.

----------------------------------------------------------------------

1. Base URL

/api/v1

Vi du: POST /api/v1/auth/login

----------------------------------------------------------------------

2. Authentication

Tat ca endpoints, tru cac public auth endpoints, yeu cau dang nhap.

Public auth endpoints:
- POST /auth/register
- POST /auth/login
- GET /auth/google/start
- GET /auth/google/callback
- POST /auth/google/mobile

Blaie dung JWT access token ngan han va refresh token stateful.

Transport theo client:

2.1. Web

- Web V1 dung HttpOnly cookies.
- Backend set access token cookie va refresh token cookie sau login/register/Google callback.
- Browser khong doc token bang JavaScript.
- Web FE goi API voi credentials/cookies.
- Backend doc token tu cookie truoc khi xu ly request.

2.2. Mobile

- Mobile dung bearer access token.
- Mobile nhan accessToken va refreshToken trong JSON response.
- Mobile gui:

  Authorization: Bearer <access_token>

- Backend doc bearer token tu Authorization header.

2.3. Backend token resolution

Backend auth filter ho tro ca hai nguon token:

1. HttpOnly cookie token cho web.
2. Authorization bearer token cho mobile.

Neu request co ca cookie va bearer token, backend nen uu tien Authorization header de
tranh nhap nhang trong cac client mobile/test.

Neu token thieu, het han, bi revoke, hoac khong hop le, API tra ve 401 UNAUTHORIZED.

Refresh token:

- Access token khong luu DB.
- Refresh token chi luu hash trong PostgreSQL.
- Web refresh token nam trong HttpOnly cookie.
- Mobile refresh token nam trong secure storage phia client.

----------------------------------------------------------------------

3. Request Format

JSON request body:
  Content-Type: application/json

File upload:
  Content-Type: multipart/form-data

Web cookie requests:
  Browser must send credentials/cookies.

Mobile bearer requests:
  Authorization: Bearer <access_token>

----------------------------------------------------------------------

4. Response Format

Tat ca response co body deu co format thong nhat:

4.1. Success co data:

{
  "data": { ... },
  "message": "optional human-readable message",
  "meta": {
    "nextCursor": "cursor_string_or_null",
    "hasMore": true,
    "limit": 20
  }
}

Ghi chu:
- "message" chi tra ve khi can thong bao bo sung.
- "meta" chi tra ve trong response danh sach.

4.2. Success khong co body:

HTTP 204 No Content - khong co response body.

4.3. Error:

{
  "code": "ERROR_CODE",
  "message": "Human-readable message cho developer/user",
  "errors": {
    "fieldName": ["Validation error message"]
  },
  "requestId": "req_abc123"
}

Ghi chu:
- "errors" chi xuat hien khi loi validation.
- "requestId" co trong moi response loi de trace log.

----------------------------------------------------------------------

5. Pagination - Cursor-based

Blaie dung cursor-based pagination cho tat ca list endpoints.

Request query params:
  cursor=<cursor_value>    optional, bo qua cho trang dau
  limit=<number>           optional, default 20, max 50

Response meta:
  nextCursor: null neu khong con trang tiep theo
  hasMore: true/false
  limit: so luong item dang dung

Vi du request trang dau:
  GET /api/v1/inbox?limit=20

Vi du request trang tiep:
  GET /api/v1/inbox?cursor=eyJpZCI6Ijg1In0&limit=20

----------------------------------------------------------------------

6. HTTP Status Codes

200  OK                    - Request thanh cong, co response body.
201  Created               - Tao resource moi thanh cong.
204  No Content            - Thanh cong, khong co body.
400  Bad Request           - Request malformed, JSON sai cu phap.
401  Unauthorized          - Chua dang nhap hoac token khong hop le.
403  Forbidden             - Da dang nhap nhung khong co quyen.
404  Not Found             - Resource khong ton tai.
409  Conflict              - Xung dot voi state hien tai.
422  Unprocessable Entity  - Validation loi.
500  Internal Server Error - Loi server khong mong muon.
503  Service Unavailable   - Server hoac dependency tam thoi khong san sang.

----------------------------------------------------------------------

7. Common Error Codes

UNAUTHORIZED              - Token thieu hoac khong hop le.
FORBIDDEN                 - Khong co quyen thuc hien action nay.
NOT_FOUND                 - Resource khong tim thay.
VALIDATION_ERROR          - Input khong hop le.
CONFLICT                  - Trang thai hien tai khong cho phep action.
INVALID_CREDENTIALS       - Sai username/email hoac password.
EMAIL_ALREADY_EXISTS      - Email da duoc dung.
USERNAME_ALREADY_EXISTS   - Username da duoc dung.
GOOGLE_AUTH_FAILED        - Google sign-in khong hop le hoac verify that bai.
INTERNAL_SERVER_ERROR     - Loi server khong xac dinh.

Error code dac thu cua tung endpoint se duoc dinh nghia trong file contract tuong ung.

  "transcriptionText": "string | null",
  "category": "task | note | calendar_event | reminder | information | null",
  "aiConfidence": 0.95,
  "title": "string | null",
  "summary": "string | null",
  "content": "string | null",
  "processingStatus": "pending | processing | completed | failed",
  "reviewStatus": "unreviewed | reviewed",
  "processingError": "string | null",
  "tags": [
    { "id": "uuid", "name": "string" }
  ],
  "taskMetadata": {
    "dueDate": "ISO8601 | null",
    "priority": "low | medium | high | null",
    "completedAt": "ISO8601 | null"
  },
  "reminderMetadata": {
    "reminderTime": "ISO8601 | null",
    "completedAt": "ISO8601 | null"
  },
  "calendarMetadata": {
