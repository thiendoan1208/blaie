API Contract - 03. Inbox (phase 1)

Base path: /api/v1/inbox

Inbox hien thi cac record con da duoc AI tach thanh cong tu raw capture cua chinh nguoi dung dang dang nhap.
Capture dang processing/failed duoc theo doi qua Capture API; capture_items chi la output phan loai thanh cong.
Tat ca truy van bat buoc filter theo user_id tu authentication; admin khong co bypass mac dinh.
Inbox read endpoints van hoat dong khi asynchronous capture acceptance bi tat (vi du drain mode). Tat acceptance
chi chan submit/retry moi; no khong an cac CaptureItem da completed cua current user.

----------------------------------------------------------------------

GET /api/v1/inbox

Mo ta: Lay cac Inbox item cua current user, moi nhat truoc. Mot text capture co the tao nhieu item trong list nay.

Query params:
  cursor - string (optional), cursor opaque tu response truoc.
  limit  - number (optional, default 20, min 1, max 50).

Response 200:
{
  "data": [
    {
      "id": "uuid",
      "originalText": "Nho goi cho me toi nay",
      "category": "reminder",
      "processingStatus": "completed",
      "createdAt": "2026-07-14T10:00:00Z"
    }
  ],
  "meta": {
    "nextCursor": "opaque cursor | null",
    "hasMore": false,
    "limit": 20
  }
}

----------------------------------------------------------------------

GET /api/v1/inbox/items/{itemId}

Mo ta: Lay mot Inbox item cua current user.

Response 200:
{
  "data": {
    "id": "uuid",
    "originalText": "Nho goi cho me toi nay",
    "category": "reminder",
    "processingStatus": "completed",
    "createdAt": "2026-07-14T10:00:00Z"
  }
}

Errors:
  404  CAPTURE_ITEM_NOT_FOUND - item khong ton tai hoac khong thuoc current user.
  422  VALIDATION_ERROR - cursor hoac limit khong hop le.

Khong co PATCH, confirm, retry hay DELETE item trong phase nay. Retry thuc hien tren Capture API.
