C4 LEVEL 3 - COMPONENT DIAGRAMS

Purpose
-------
This document zooms into the two most important containers in MVP V1:

- Backend API
- AI Processing Worker

This is not a code diagram. It shows modules/components and responsibilities, not
classes/interfaces.

Auth Module
- Handles signup, login, logout and user identity.

Authorization Layer
- Ensures users can only access their own data.
- Separates normal user permissions from admin permissions.

Capture Module
- Core V1 component.
- Accepts text/image/audio capture requests.
- Validates basic input.
- Saves item with processing status pending.
- Calls File Upload Module if files are included.
- Calls Processing Job Publisher.
- Returns response quickly without waiting for AI.

File Upload Module
- Coordinates image/audio upload.
- Stores file references.

Item Management Module
- Handles item detail, manual edits, category changes, metadata updates, archive/delete
  and review status.
- Manual user edits must override AI suggestions.

Inbox/View Query Module
- Provides read models for Inbox, Tasks, Calendar events, Reminders and
  Information items.

Search & Tag Module
- Supports basic search and tag filtering in V1.

Reminder Module
- Manages reminder state.
- Prepares notification requests.

Admin Module
- Allows admin to inspect processing jobs, retry failed jobs, view metrics and audit
  admin actions.

Processing Job Publisher
- Creates background processing jobs after capture or retry request.

Repository Layer
- Data access boundary for PostgreSQL.

Object Storage Adapter
- Encapsulates file storage access.
- Keeps backend business logic independent from a specific storage provider.

Job Consumer
- Reads pending jobs from the queue.
- Starts background processing.

Processing Status Manager
- Updates item/job status:
  - pending
  - processing
  - completed
  - failed

Processing Orchestrator
- Chooses pipeline based on source type:
  - text
  - image
  - voice/audio

Text Processor
- Prepares text input for classification and metadata extraction.

Image Processor
- Extracts useful text or meaning from image input using OCR and/or image understanding.

Audio Processor
- Transcribes voice/audio input before classification.

AI Provider Adapter
- Encapsulates calls to external AI provider.
- Makes future provider/model changes easier.

Classification Component
- Classifies item into exactly one V1 category:
  - task
  - calendar_event
  - reminder
  - information

Metadata Extraction Component
- Extracts title, summary, due date, reminder time, event start/end time, priority, tags,
  confidence and reason.

Result Writer
- Persists AI output to PostgreSQL.
- Preserves original input.
- Must not overwrite user-edited fields unless user explicitly requests reprocessing.

Retry & Error Handler
- Handles failed AI/OCR/transcription jobs.
- Stores technical failure reason for admin debugging.

Notification Dispatcher
- Sends reminder notifications when needed and when permission/platform support exists.


Component design principles
---------------------------

- Capture path must be short and fast.
- AI path must be asynchronous and retryable.
- User data ownership must be checked in backend, not only in frontend.
- Manual edits must win over AI suggestions.
- Original input and AI output should be stored separately.
- Admin tooling should inspect job status and errors without exposing user content unless
  explicitly required for debugging.
